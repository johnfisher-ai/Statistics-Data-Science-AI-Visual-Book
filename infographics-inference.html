01

Point & Interval Estimation

Chapters 69–70
In a nutshell
  • A point estimate (like x̄) is a single best guess; it wobbles from sample to sample by the standard error, SE = σ/√n.
  • An interval estimate wraps the point in honest uncertainty: estimate ± margin. A 95% interval is built so the procedure covers the truth about 95% of the time.
  • Use the z-interval when σ is known (or n is large) and the t-interval when σ is estimated from a small sample.

Confidence Intervals & Margin of Error

Chapters 71–73
In a nutshell
  • The margin of error is z·SE, the half-width of the interval; it grows with confidence and variability and shrinks like 1/√n (quadruple n to halve it).
  • Intervals exist for a mean, a proportion, and a difference; when no clean formula exists (a median, a correlation), the bootstrap resamples the data to build one.
  • "95% confident" describes the method over many samples, not a probability about one particular interval.

The Logic of Hypothesis Testing

Chapter 74
In a nutshell
  • State H₀ (no effect, the skeptic's default) and H₁ (the claim); choose one-sided or two-sided before seeing the data.
  • Pick a test statistic, build its distribution assuming H₀ is true, then see how far into the tail the observed value falls.
  • The p-value is the chance of data this extreme if H₀ were true (not the probability that H₀ is true); reject H₀ when p < α.

04

Significance, Errors & Power

Chapter 75
In a nutshell
  • α is the Type I (false-alarm) rate you accept; β is the Type II (miss) rate; power = 1 − β is the chance of catching a real effect.
  • Power rises with sample size and effect size; "no significant difference" from a small, underpowered study is nearly uninformative.
  • A p-value is not an effect size: with huge n a trivial effect is "significant", so always report an effect size and an interval too.

05

The Test Family: z · t · F · χ²

Chapters 76–79
In a nutshell
  • z-tests use the normal curve for large-sample means and for proportions (one-sample uses p₀ for the SE; two-sample pools the rate).
  • t-tests compare one mean, two independent groups (Welch), or paired data; ANOVA (F) compares 3+ groups at once, then Tukey HSD finds which pairs differ.
  • Chi-square compares observed to expected counts (goodness-of-fit and independence), with Cramer's V as the effect size.

06

Nonparametric & Choosing the Right Test

Chapters 80–81
In a nutshell
  • Four questions pick the test: outcome type (numeric vs categorical), number of groups, independent vs paired, and whether the assumptions hold.
  • Nonparametric tests work on ranks, so they tolerate skew, outliers, and ordinal data: Mann-Whitney, Kruskal-Wallis, and Wilcoxon are the twins of the t-test, ANOVA, and paired t.
  • The trade-off is a little less power when the data really are normal, but they stay valid when they are not.

07

A/B Testing & Online Experiments

Chapter 82
In a nutshell
  • Randomly assign users to A or B so the groups match on everything but the change; fix the sample size first from a power analysis (n grows like 1/MDE²).
  • Analyze the primary rate with a two-proportion z-test and a confidence interval for the lift; report the interval, not just the p, and watch guardrail metrics.
  • Do not peek: repeatedly checking and stopping at p < 0.05 turns a 5% test into roughly a 25% false-positive rate; analyze once, or use sequential methods.

08

The Statistician's Workflow

Chapters 83–88
In a nutshell
  • The professional workflow is a pipeline: explore the data, choose the test, state H₀/H₁, run it with an effect size, then report in plain language for a non-statistician.
  • One dataset can answer several questions, each needing a different test (paired, two-sample, proportion), and a healthy-looking aggregate can hide a real subgroup problem, so segment then test.
  • Use the standard libraries (scipy and statsmodels, the Python answer to R and SAS) so the software does the standard errors and you focus on the decision.
