What Is Machine Learning?

Chapter 104
In a nutshell
  • Machine learning flips the classic recipe: instead of coding the rules, you feed the machine examples and it learns the rules that map inputs to outputs.
  • Three paradigms: supervised (learn from labeled examples to predict a label or number), unsupervised (find structure in unlabeled data), and reinforcement (learn a policy from rewards).
  • ML and statistics overlap heavily; the emphasis differs, statistics prizes explanation and inference, ML prizes out-of-sample prediction.

The ML Workflow & Data Leakage

Chapter 105
In a nutshell
  • Split the data into training, validation, and test: tune on the first two, and touch the test set once at the very end for an honest read.
  • Cross-validation rotates a held-out fold through the training data and averages the scores, a more stable estimate than a single split, and the basis for tuning.
  • Data leakage is the cardinal sin: any transform (scaling, imputing, feature-building) fit on the full data before splitting lets the test set peek, inflating the score. Wrap steps in a Pipeline so they fit on training folds only.

The Algorithm Chooser

Chapters 106–107
In a nutshell
  • Algorithms differ in the shape of boundary they draw: logistic regression and linear SVM cut a straight line, KNN follows the local crowd, and a decision tree makes axis-aligned box splits.
  • Start with a simple, interpretable baseline (logistic or a shallow tree); reach for trees/KNN when the pattern is nonlinear and for ensembles when you need raw accuracy.
  • Distance-based methods (KNN, clustering, SVM kernels) depend on a distance metric, Euclidean, Manhattan, cosine, Jaccard, so scale your features first or the large-range ones dominate.

Ensembles: Bagging, Boosting & Stacking

Chapter 108
In a nutshell
  • Bagging (and random forests) trains many trees in parallel on bootstrap samples and averages their votes, mainly cutting variance.
  • Boosting trains weak learners in sequence, each correcting the last one's errors, mainly cutting bias; stacking trains a meta-model to blend several different models.
  • The payoff is real: a single deep tree overfits (about 0.70 here) while an ensemble generalizes (about 0.79), which is why forests and gradient boosting win so many tabular problems.

Overfitting & the Bias-Variance Tradeoff

Chapters 109–110
In a nutshell
  • Prediction error splits into bias (too simple, underfits) and variance (too complex, overfits the noise); the goal is the balance between them, not zero of either.
  • As model complexity grows, training error keeps falling but test error is U-shaped, a widening gap between the two is the signature of overfitting.
  • Regularization (L1/L2 penalties), more data, and fewer features pull you toward the sweet spot; gradient descent is the engine that actually finds the weights by stepping downhill on the loss.

Evaluating a Model: Metrics & the Confusion Matrix

Chapters 111–113
In a nutshell
  • The confusion matrix (true/false positives and negatives) is the source of every classification metric; precision asks “of those flagged, how many were right?” and recall asks “of the real cases, how many did we catch?”
  • Accuracy is a trap on imbalanced data (predicting the majority scores high while helping no one); pick the metric, and the threshold, that matches the decision's costs, and use ROC-AUC / PR-AUC to judge ranking.
  • Evaluation is not just a number: watch for pitfalls (leakage, wrong metric, unstable folds) and add interpretability (SHAP) so each prediction comes with a reason.

Unsupervised & Reinforcement Learning

Chapters 114–119
In a nutshell
  • Unsupervised learning works without labels: clustering groups similar rows, PCA compresses many features into a few, association rules find “what goes with what”, and anomaly detection flags the odd ones out.
  • Reinforcement learning has an agent take actions in an environment and learn a policy that maximizes long-run reward, balancing explore vs exploit.
  • From Q-learning to deep RL and RLHF, it is the paradigm behind game-playing agents and the alignment fine-tuning of large language models.

The ML Project: from a Model to Production

Chapters 120–126
In a nutshell
  • Every ML project follows one repeatable 12-step method, framed in five phases from defining the problem through deploying and monitoring, applied here across seven case studies (fraud, maintenance, segmentation, loan risk, and more).
  • The hard judgement lives around the model: choosing the metric, thresholding by cost, avoiding leakage, checking fairness, and communicating the result to a decision-maker.
  • A deployed model is on the clock: drift silently decays it, so production is a never-ending loop of monitor, detect drift, retrain, and roll out, the loop, not the one-time model, is the deliverable.
